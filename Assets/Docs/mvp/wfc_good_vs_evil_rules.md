# WFC Rules Ideas for Good vs Evil Architecture

## Design Principle for WFC Rules

Do NOT randomize everything.

Good WFC systems rely on: - constrained grammar - repeated motifs -
predictable structure logic - controlled variation

The structure should feel intentional.

------------------------------------------------------------------------

# Core Structural Axes of Differentiation

Use these 5 axes to create alignment identity:

1.  Verticality
2.  Density
3.  Symmetry
4.  Curvature vs sharpness
5.  Light openness vs enclosed mass

------------------------------------------------------------------------

# GOOD Alignment --- WFC Rule Ideas

## Visual Theme

order, elevation, light, openness, elegance

## Shape Grammar

-   taller structures more likely
-   vertical stacking probability increases
-   arches frequently appear between blocks
-   curved connectors preferred
-   towers generate naturally from clusters

## Example adjacency rules

-   pillars prefer adjacent pillars vertically
-   arches prefer horizontal connections
-   roofs generate after 3+ vertical layers
-   open courtyards appear after 4-block clusters
-   bridges appear when 2 elevated nodes connect

## Structure motifs

-   cathedral-like silhouettes
-   floating platforms
-   ring structures
-   spires
-   terraces

## Emergent geometry bias

-   more empty space inside structures
-   hollow towers
-   outward branching
-   soft slopes

## Gameplay tie-ins

possible unlocks: - glide boost towers - floating resource nodes -
visibility buffs - faster travel routes

## Emotional effect

feels uplifting feels safe feels expansive

------------------------------------------------------------------------

# EVIL Alignment --- WFC Rule Ideas

## Visual Theme

compression, weight, corruption, tension

## Shape Grammar

-   horizontal spread preferred
-   high density clustering
-   spikes appear probabilistically on edges
-   enclosed volumes preferred
-   underground expansion nodes

## Example adjacency rules

-   blocks prefer multiple side attachments
-   spikes appear when exposed edges detected
-   thicker walls emerge after 3+ horizontal neighbors
-   negative space minimized
-   branching creates tangled geometry

## Structure motifs

-   fortresses
-   thorn-like protrusions
-   ribbed walls
-   nested chambers
-   jagged silhouettes

## Emergent geometry bias

-   dense interiors
-   inward curvature
-   layered shells
-   asymmetry

## Gameplay tie-ins

possible unlocks: - mob spawning chambers - corruption resource nodes -
defensive advantages - high-risk crafting materials

## Emotional effect

feels dangerous feels powerful feels claustrophobic

------------------------------------------------------------------------

# Neutral Base Rule Set (Starting Grammar)

Both alignments derive from the same core pieces.

Starter modules: - cube room - half height wall - column - connector -
floor tile - roof cap

Alignment modifies: probabilities, adjacency permissions,
transformations.

Reuse pieces with rule bias.

------------------------------------------------------------------------

# Simple MVP Rule Implementation

Keep rule count small.

Example:

good bias: - increase probability of vertical adjacency - enable arch
transformation when 2 columns align - create balcony when exposed edge
detected

evil bias: - increase probability of lateral adjacency - add spike
variant to exposed edge - convert open face into ribbed wall

Small rule differences produce large visual divergence.

------------------------------------------------------------------------

# Visual Feedback Signals

Players should recognize alignment quickly through:

-   silhouette
-   density
-   negative space
-   height variation

Color should reinforce structure, not define it.

Example palette:

good: - white stone - teal glass - gold accents

evil: - dark basalt - red glow - bone texture

------------------------------------------------------------------------

# MVP-safe WFC rule set (minimum viable)

6 base tiles: - block - column - arch - roof - connector - spike variant

3 good rules: - vertical stacking bias - arch generation rule - balcony
generation rule

3 evil rules: - horizontal clustering bias - spike mutation rule -
enclosed chamber rule

Enough to create noticeable variation.

------------------------------------------------------------------------

# Critical warning

WFC easily becomes overengineered.

Avoid: - hundreds of tiles - complex constraints - 3D grammar explosion

Start with simple grammar producing strong silhouettes.

Complexity can emerge later.
